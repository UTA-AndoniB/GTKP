# Overview

Guardian Tales Knowledge Project (GTKP) is an open-source platform designed to centralize, organize, analyze, and optimize every relevant aspect of the game *Guardian Tales*.

The project aims to provide players with professional-grade tools to manage their accounts, optimize resources, build competitive teams, understand game mechanics, and preserve historical game knowledge.

Rather than being a simple account tracker, GTKP is designed as a modular knowledge platform capable of supporting multiple applications built on top of a shared core.

---

# Vision

Build the most complete open-source knowledge platform for Guardian Tales.

The project should allow both casual and competitive players to make informed decisions based on reliable data, reproducible calculations, and transparent recommendation engines.

---

# Mission

Develop a modular software ecosystem capable of:

- Managing player accounts
- Tracking heroes, weapons and collections
- Planning resource investments
- Building optimized teams
- Preserving historical game data
- Providing intelligent recommendations
- Supporting community contributions

---

# Core Principles

The project follows several engineering principles.

## Data First

Game data is the single source of truth.

Logic must never duplicate game information.

---

## Modular Design

Every system must be independent.

Modules should be replaceable without affecting the rest of the project.

---

## Open Source

All documentation, code, schemas and datasets should be publicly accessible.

---

## Reproducibility

Every recommendation generated by GTKP must be reproducible.

The user should always understand why a recommendation was generated.

---

## Maintainability

The project must prioritize readability over clever implementations.

---

## Extensibility

New heroes, weapons, relics or future game systems should be added without redesigning the architecture.

---

# Project Layers

GTKP is organized into five major layers.

```
Community
↑
Interfaces
↑
Adapters
↑
Core
↑
Data
```

Each layer has a single responsibility.

---

## Data

Stores all game information.

Examples:

- Heroes
- Weapons
- Collections
- Merch
- Relics
- Cards

---

## Core

Contains the business logic.

Examples:

- Hero
- Weapon
- Inventory
- Recommendation
- Team

---

## Adapters

Translate external data into Core objects.

Examples:

- Google Sheets
- CSV
- JSON
- GitHub

---

## Interfaces

User interaction.

Examples:

- Google Sheets
- Website
- Dashboard
- API

---

## Community

Knowledge generated by players.

Examples:

- Tier Lists
- Team Builds
- Guides
- Meta Reports
